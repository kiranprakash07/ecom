To understand how this e-commerce application works, think of it as a restaurant: HTML is the structural layout (the tables and chairs), CSS/Bootstrap is the interior design and presentation, and JavaScript is the waitstaff and kitchen handling all the logic and movement.

Here is the step-by-step breakdown of how these three files interact to create your store.

Step 1: Laying the Structural Foundation (index.html)
The HTML file acts as a static blueprint. Instead of hardcoding 20 separate product cards manually (which would make the file thousands of lines long), we created empty structural "containers" that JavaScript can fill later.

The Grid Anchor: Inside the <main> tag, we created a single div with an ID:

HTML
<div id="product-grid" class="row ..."></div>
This is an empty bucket. It tells the browser exactly where the products should appear on screen.

The Sidebar Cart: We built a container hidden off-screen (transform: translateX(100%)) that holds an empty item list (#cart-items) and a checkout button.

The Badges: The shopping bag icon contains a small <span> tag (#cart-count) with the class d-none (Bootstrap for display: none) so it stays hidden until an item is added.

Step 2: Styling and Responsiveness (style.css & Bootstrap)
Bootstrap handles the heavy lifting for the responsive grid layout, while the custom CSS manages smooth animations.

The Responsive Grid: In HTML, we gave our grid container these Bootstrap classes: row-cols-1 row-cols-sm-2 row-cols-md-3 row-cols-lg-4. This automatically instructs the browser:

Show 1 column on mobile screens.

Show 2 columns on tablets (sm).

Show 3 columns on small laptops (md).

Show 4 columns on large desktop monitors (lg).

The Sliding Cart: In style.css, the .cart-sidebar is fixed to the right side of the screen. When JavaScript adds the class .open, CSS smoothly transitions the sidebar into view using transform: translateX(0).

Step 3: Modeling the Data (script.js)
At the very top of your JavaScript file, we created a central database using a List of Objects called products:

JavaScript
const products = [
    { id: 1, name: "Premium White Tee", price: 29.99, image: "..." },
    ...
];
Every single product has a unique ID, a name, a decimal price, and an image URL. This makes it incredibly easy to manage; if you want to change a price or add a 21st product, you only modify this array, not the HTML structure.

Step 4: Injecting the Products (displayProducts())
When the browser finishes loading the files, the JavaScript function displayProducts() runs automatically.

It targets the empty HTML bucket using document.getElementById('product-grid').

It uses the .map() function to loop through all 20 items in your array.

For every single item, it generates a block of HTML code (a product card) and inserts the item's unique image, name, and price dynamically using template literals (${product.name}).

Crucially, it attaches an event listener to the button: onclick="addToCart(${product.id})". This ensures every button knows exactly which product ID it belongs to.

Step 5: Managing the State (addToCart() & updateCartUI())
When a user clicks an "Add to Cart" button, a chain reaction occurs:

[Click Button] ➔ [addToCart(id)] ➔ [Update Array] ➔ [updateCartUI()]
Check for existence: addToCart(productId) looks inside a second array called cart.

Logic Branch: * If that product ID is already in the cart, it simply increases its quantity by 1.

If it is a new item, it pushes the entire product object into the cart array with a starting quantity: 1.

Redraw the UI: Finally, it calls updateCartUI(). This function wipes out whatever was previously inside the sidebar cart, recalculates the total price, updates the red navbar counter badge, and completely redraws the current list of cart items so the user sees their updated totals instantly.
